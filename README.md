# RevenuePilot AI

AI-CRM, которая не просто хранит клиентов, а помогает не терять лидов и продавать
больше. Лиды приходят сами через Telegram-бота, менеджеры общаются с клиентами через
бота, а AI всё время следит за перепиской, анализирует и подсказывает следующий шаг.

> Статус: бэкенд реализован (`backend/`) — FastAPI + PostgreSQL по `openapi.yaml`.
> Фронтенд инициализирован (`frontend/`); подключение к API — следующий шаг.

## В двух словах

- **Источник лидов** — Telegram-бот. Клиент пишет боту → автоматически создаётся лид.
- **Канал общения** — под каждого клиента бот создаёт отдельный топик в рабочей
  Telegram-группе менеджеров. Менеджеры отвечают в топике, бот пересылает ответ
  клиенту от своего имени.
- **AI внутри** — в реальном времени считает вероятность покупки, риск ухода,
  делает саммари клиента и говорит, что делать прямо сейчас.

## Как это работает

```
Клиент → Telegram-бот → топик в группе менеджеров → CRM + AI
                ▲                                        │
                └──────── ответ бота клиенту ◄───────────┘
```

1. Клиент пишет боту.
2. Бот заводит лид и создаёт под него топик в форум-группе.
3. Сообщения клиента падают в топик и в CRM.
4. Менеджер отвечает в топике (или в веб-CRM) — клиент получает ответ от бота.
5. AI на каждом сообщении пересчитывает score, риск, саммари и next best action.

## Возможности

- 🤖 **Telegram как канал** — единый бот, изоляция клиентов по топикам.
- 📊 **AI-анализ лидов** — вероятность покупки, риск ухода, заинтересованность, причины.
- 👻 **Anti-Ghost** — находит забытых и остывающих клиентов, напоминает дожать.
- ✍️ **AI-генерация ответов** — персональные сообщения с выбором тона.
- 🧠 **AI-саммари клиента** — выжимка вместо чтения всей переписки.
- 🛠️ **AI-воронки** — строит воронку под тип бизнеса и показывает узкие места.
- 💬 **AI-ассистент** — чат-вопросы по базе («кому написать сегодня?»).
- 🚀 **«Как увеличить продажи?»** — AI находит упущенную выручку и готовый план действий.

## Команды бота (для менеджеров в топике)

| Команда            | Что делает                                          |
| ------------------ | --------------------------------------------------- |
| `/ai_reply`        | Сгенерировать ответ клиенту по всей переписке       |
| `/ai_summary`      | Короткая выжимка по клиенту                          |
| `/ai_score`        | Вероятность покупки, риск ухода, причины             |
| `/ai_next`         | Лучшее следующее действие                            |
| `/ai_tone <тон>`   | Ответ в заданном тоне (дружелюбно/уверенно/коротко)  |

## Аутентификация менеджеров

Менеджеры заходят в веб-CRM по логину/паролю и получают JWT
(access + refresh токены). Все защищённые эндпоинты требуют
`Authorization: Bearer <access_token>`. Подробности — в `openapi.yaml`.

## Стек

- **Frontend:** React 19 + Vite + TypeScript (веб-CRM, уже инициализирован).
- **Backend:** FastAPI + SQLAlchemy 2.0 (async) + Alembic + PostgreSQL по `openapi.yaml`,
  JWT-аутентификация, Telegram Bot API (webhook). См. [`backend/`](backend/).
- **AI:** LLM для анализа переписки, генерации ответов и инсайтов с детерминированным
  эвристическим фолбэком (работает и без LLM-ключа).

## Быстрый старт бэкенда

```bash
cd backend
docker compose up -d db
uv venv && source .venv/bin/activate && uv pip install -e ".[dev]"
cp .env.example .env
alembic upgrade head
python -m app.seed          # демо-данные: demo@revenuepilot.ai / demo12345
uvicorn app.main:app --reload --port 8000
# Swagger UI: http://localhost:8000/api/v1/docs
```
